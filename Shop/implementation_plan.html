Chuyển đổi nguồn dữ liệu JS → MySQL (Database_Web)

Chuyển toàn bộ dữ liệu sản phẩm (đang ở products-data.js) và dữ liệu giỏ hàng (đang ở localStorage) sang lấy từ database MySQL tên Database_Web, thông qua một backend API trung gian.

User Review Required

Cần có backend: Trình duyệt (JS) không thể kết nối trực tiếp đến MySQL. Bắt buộc phải có một backend server (Node.js + Express) làm trung gian giữa frontend và database. Plan này sẽ tạo thêm folder backend/ trong Shop/.
Phụ thuộc mới: Cần cài đặt node.js, npm, và các package express, mysql2, cors. Các máy chạy dự án đều cần có Node.js.

Các quyết định đã xác nhận

# Câu hỏi Quyết định
1 Bảng users Chưa có → dùng user_id giả định (hardcode user_id = 1 trong frontend). Khi có bảng users sẽ bổ sung FK sau.
2 Port MySQL Sử dụng port mặc định 3306. Backend API chạy ở port 3000.
3 Xác thực user Chưa có hệ thống xác thực → frontend truyền user_id trực tiếp qua API request. Khi có login sẽ thay bằng session/JWT sau.

Tóm tắt luồng dữ liệu sau chuyển đổi

1. User mở trang Shop
   → Product Card.js gọi GET /api/products
   → Backend query: SELECT * FROM product_data
   → Trả về JSON → render sản phẩm

2. User bấm "Add to cart"
   → Product Card.js gọi POST /api/cart { userId, productId, quantity }
   → Backend: INSERT INTO cart ... ON DUPLICATE KEY UPDATE quantity
   → Giỏ hàng lưu trong MySQL, đồng bộ mọi thiết bị

3. User mở trang Cart
   → cart.js gọi GET /api/cart/:userId
   → Backend query: SELECT c.*, p.name, p.image_url, p.current_price
                     FROM cart c JOIN product_data p ON c.product_id = p.product_id
                     WHERE c.user_id = ?
   → Trả về JSON → render giỏ hàng
